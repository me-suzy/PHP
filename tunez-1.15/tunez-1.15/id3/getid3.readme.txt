What does getID3() do?
======================

Reads & parses (to varying degrees):
  * AIFF
  * APE tags: v1 and v2
  * ASF: ASF, Windows Media Audio (WMA), Windows Media Video (WMV)
  * AU
  * BMP
  * Bonk
  * CD-audio (*.cda)
  * FLAC
  * Flash
  * GIF
  * ID3v1 & ID3v1.1
  * ID3v2.4, ID3v2.3, ID3v2.2
  * ISO-9660 CD-ROM image (directory structure)
  * JPEG
  * LA (Lossless Audio)
  * LPAC
  * Lyrics 3: v1 & v2
  * Lyrics3 v1 & v2
  * MIDI
  * Monkey's Audio
  * MP3/MP2/MP1
  * MPC / Musepack
  * MPEG video
  * NSV (Nullsoft Streaming Video)
  * Ogg (Vorbis, OggFLAC, Speex)
  * OptimFROG
  * PNG
  * Quicktime
  * RealAudio, RealVideo
  * RIFF: AVI/WAV
  * Speex
  * VOC
  * VQF
  * WavPack
  * ZIP (directory structure)

Writes:
  * ID3v1 & ID3v1.1
  * ID3v2.3 & ID3v2.4
  * Ogg Vorbis comment tags
  * APEv2


Requirements
============

* PHP 4.1.0 (or higher)
* GD  <1.6 for GIF and JPEG functions
* GD >=1.6 for PNG and JPEG functions


Notes
=====

If the format parser encounters a problem, it will return something
in $fileinfo['error'], describing the encountered error. If nothing is
returned in that array element, you can assume the entire file parsed OK.

Conforms to ID3v2.2, ID3v2.3 and ID3v2.4 specs as published at www.id3.org


About AssumeFormat
==================

getID3() can indeed identify almost all corrupted MP3s of various
kinds, but to do so it has to scan more agressively, and assume that
the file is in MP3 format, rather than objectively examining the
beginning of the data stream to see what format the file appears to
be in.

Take a look at getid3.check.php and you will notice (somewhere around
lines 42-48) how the AssumeFormat parameter of GetAllFileInfo() is
used: The file is first scanned assuming nothing, but if that comes
back with no information on the file, it is re-scanned assuming that
the file extension is in fact correct (so if "something.mp3" comes
back as unknown format, try scanning it again assuming it is really
an MP3 file) which scans more aggressively and can usually identify
just about any kind of "broken" MP3 file.

Usage
=====

For detailed examples of implementation, see getid3.demo.check.php

See getid3.demo.scandir.php for a sample recursive scanning code that
scans every file in a given directory, and all sub-directories

See getid3.demo.simple.php for a simple example script that scans all
files in one directory and output artist, title, bitrate and playtime


$mp3info = GetAllFileInfo(<filename>);
$mp3info = GetAllFileInfo('/home/mp3s/song.mp3');
$mp3info = GetAllFileInfo('c:\\mp3s\\song.mp3');


What does the returned data structure look like?
================================================

See getid3.structure.txt

It is recommended that you look at the output of getid3.demo.check.php scanning
the file(s) you're interested in to confirm what data is actually returned for
any particular filetype in general, and your files in particular, as the actual
data returned may vary considerably depending on what information is available
in the file itself.
